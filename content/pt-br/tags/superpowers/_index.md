Superpowers é um plugin pro Claude Code que adiciona três coisas concretas ao agent: skills (markdown que descreve como fazer X), agents/subagents (invocações especializadas com contexto próprio), e slash commands (atalhos que disparam fluxos complexos).

Soa parecido com prompts salvos? É. Mas a diferença não está no conteúdo — está no ritual. Skill enforced significa que o Claude lê a skill ANTES de começar a tarefa. Não tem chance de pular TDD. Não tem chance de pular brainstorming. Não tem chance de afirmar "feito" sem rodar verification. O hábito do dev sênior vira regra executada pela máquina.

Não é mais um plugin de IA. É a forma de codificar conhecimento de time num lugar reprodutível. Onde antes "o jeito de trabalhar" morava na cabeça de quem estava há mais tempo no time, agora mora em arquivo markdown que qualquer dev novo invoca via slash command e ganha o mesmo padrão.

## Por que este tópico importa

O ângulo Nextside: doc de processo é ficção, skill executada é processo de verdade. Confluence ninguém lê depois da segunda semana. Skill no Claude executa toda vez que a tarefa começa. A diferença é entre processo passivo e processo ativo.

Reprodutibilidade entre devs, fim do prompt artesanal escrito às 3 da manhã, TDD e brainstorming forçados quando o cérebro humano já está cansado demais pra lembrar. Superpowers não é alavanca de velocidade — é alavanca de consistência. E em time pequeno entregando paralelo, consistência vale mais que velocidade bruta.
