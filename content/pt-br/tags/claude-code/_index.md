Claude Code é a CLI agentic da Anthropic. Roda no terminal, lê o repositório inteiro, edita arquivos, executa comandos, abre PR. Não é Copilot melhorado. Copilot completa linha. Claude Code conduz tarefa.

A diferença não é capacidade bruta — é onde a IA opera. Autocomplete vive dentro do editor, num contexto de 200 linhas. Agent vive no terminal, com acesso ao shell, ao filesystem e ao histórico do projeto. Isso muda o tipo de trabalho que cabe pra IA: deixa de ser "termina essa função" e vira "implementa esse plano e me reporta".

E aí mora a armadilha. Sem ritual, agent vira gerador de slop convicto — código que parece certo mas erra no detalhe que importa. Com ritual codificado em skills, agents e slash commands, o mesmo Claude Code entrega trabalho com disciplina de dev sênior cansado.

Os posts marcados com `claude-code` cobrem o que a gente da Nextside aprendeu rodando IA agentic em projeto real — não em demo de palco. Construindo design system, escrevendo conteúdo, automatizando UX review, integrando MCP. O que funciona, o que falha, o que economiza semana e o que custa semana.

## Por que este tópico importa

O ângulo Nextside é simples: IA boa não substitui processo. Codifica o processo num lugar que a máquina lê toda vez — `CLAUDE.md`, skills, agents, hooks — e o resultado deixa de depender de você lembrar de pedir teste, brainstorming, verification. O processo executa sozinho.

Quem fica no vibe coding solto entrega rápido e refaz devagar. Quem codifica o jeito de trabalhar em skill executa o mesmo padrão toda sessão. É a diferença entre ter IA no time e ter time com IA.
